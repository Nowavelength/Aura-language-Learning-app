### Path Interpolation Strategy

The cleanest approach for morphing is **Flubber.js** (SVG path morphing) or a direct numeric lerp if you control the point count. Since all paths above use cubic Bézier control structures with the **same number of command segments per shape class**, you can implement a pure JS lerp:

```typescript
// Each path string → array of [cmd, ...args] tuples
function parsePath(d: string): number[] {
  // extract all numeric tokens, leaving command letters aside
  return d.match(/-?[\d.]+/g)!.map(Number);
}

function lerpPath(
  pathA: string, pathB: string, t: number
): string {
  const a = parsePath(pathA);
  const b = parsePath(pathB);
  // Linear interpolate every coordinate
  const interp = a.map((v, i) => v + (b[i] - v) * t);
  // Re-template into the canonical command structure
  return rebuildPath(interp); // replace numerics back into command string
}

// Usage with requestAnimationFrame or CSS custom properties:
// element.setAttribute('d', lerpPath(tongueNeutral, tongueHighFront, easeInOut(t)));
```

For the **velum**, because the raised and lowered paths have the same command structure (`M ... Q ... Q ... Q ...`), numeric lerp is perfectly sufficient. Smoothly animate `t` from `0` → `1` over ~120ms for a realistic velopharyngeal closure gesture. For the **trill oscillation** on the tongue tip circle, use a sinusoidal offset directly (see Output 2 formula below).

```typescript
import * as THREE from 'three';

// Assume your GLTF has morph targets named to match the parameter keys
function applyArticulatoryWeights(
  mesh: THREE.SkinnedMesh,
  weights: ArticulatoryWeights,
  morphIndexMap: Record<keyof ArticulatoryWeights, number>
): void {
  if (!mesh.morphTargetInfluences) return;
  for (const [param, value] of Object.entries(weights)) {
    const idx = morphIndexMap[param as keyof ArticulatoryWeights];
    if (idx !== undefined) {
      mesh.morphTargetInfluences[idx] = value;
    }
  }
}

// Smooth transition between two articulation targets
function lerpWeights(
  a: ArticulatoryWeights,
  b: ArticulatoryWeights,
  t: number
): ArticulatoryWeights {
  const lerp = (x: number, y: number) => x + (y - x) * t;
  return {
    JA:  lerp(a.JA,  b.JA),
    TBH: lerp(a.TBH, b.TBH),
    TBB: lerp(a.TBB, b.TBB),
    TTE: lerp(a.TTE, b.TTE),
    VEL: lerp(a.VEL, b.VEL),
    LIR: lerp(a.LIR, b.LIR),
    LIP: lerp(a.LIP, b.LIP),
  };
}
```

### Spanish Alveolar Trill — Tongue Tip Flutter Mathematics

The trill is not a simple contact. It is a **myoelastic-aerodynamic oscillation**: the tongue tip is positioned near (not at) the alveolar ridge, subglottal air pressure builds, lifts the tip into contact (closure phase), pressure releases, tip elasticity snaps it back, and the cycle repeats. Physical measurements from Recasens (1991) and Solé (2002) place the fundamental frequency at **24–34 Hz**, with a modal value around **27 Hz**.

The complete mathematical model for your render loop:

```typescript
// Spanish trill tongue-tip oscillation model
// Based on myoelastic-aerodynamic two-mass oscillator (van den Berg 1958, adapted)

const TRILL_FREQUENCY_HZ = 27;           // modal trill rate
const TRILL_AMPLITUDE    = 0.45;         // fraction of TTE range (0.45 * max_tip_displacement)
const DUTY_CYCLE         = 0.38;         // contact phase fraction per cycle
                                          // (tip is sealed ~38% of each cycle)
const TRILL_ONSET_RAMP   = 0.080;        // seconds to reach full amplitude (soft onset)
const BASE_TTE           = 0.95;         // target TTE weight at rest position near alveolar

// Smoothed asymmetric oscillator:
// - Opening phase (tip retreating): cosine curve, fast
// - Closure phase: clipped flat at max (contact held)
function trillTTE(
  timeSeconds: number,
  trillAge: number    // time since trill onset, for amplitude ramp
): number {
  const ω  = 2 * Math.PI * TRILL_FREQUENCY_HZ;
  const φ  = ω * timeSeconds;             // phase
  
  // Raw sine oscillation
  const raw = Math.sin(φ);
  
  // Asymmetric shaping: clip the upper half (contact phase)
  // Values above threshold → held at ceiling (simulating airtight closure)
  const contactThreshold = Math.cos(Math.PI * DUTY_CYCLE); // ≈ 0.51
  const shaped = raw > contactThreshold
    ? 1.0
    : (raw - (-1)) / (contactThreshold - (-1)); // remap [-1, threshold] → [0, 1]
  
  // Onset amplitude envelope (ramp in)
  const envelope = Math.min(1.0, trillAge / TRILL_ONSET_RAMP);
  
  // Final TTE value: base position ± oscillation
  const oscillation = shaped * TRILL_AMPLITUDE * envelope;
  return Math.max(0, Math.min(1, BASE_TTE - TRILL_AMPLITUDE * 0.5 + oscillation));
}

// Turbulence burst on each closure release (adds acoustic realism to visuals)
// Returns 0.0–1.0 turbulence intensity, spikes on the transition from contact → open
function trillTurbulenceBurst(timeSeconds: number): number {
  const ω   = 2 * Math.PI * TRILL_FREQUENCY_HZ;
  const φ   = ω * timeSeconds;
  const raw = Math.sin(φ);
  // Differentiate: d/dt sin(ωt) = ω cos(ωt)
  // Turbulence spikes when velocity is maximum (mid opening phase)
  const velIndicator = Math.cos(φ);        // proportional to tip velocity
  return Math.max(0, velIndicator) ** 2;   // only positive half, squared for peakiness
}

// In your Three.js animation loop:
const clock = new THREE.Clock();
let trillAge = 0;
let isTrillActive = false;

renderer.setAnimationLoop(() => {
  const delta = clock.getDelta();
  const elapsed = clock.getElapsedTime();
  
  if (isTrillActive) {
    trillAge += delta;
    const currentWeights = { ...ARTICULATORY_TARGETS["es_r_trill"] };
    currentWeights.TTE = trillTTE(elapsed, trillAge);
    // Optionally: scale an airflow particle emitter by trillTurbulenceBurst(elapsed)
    applyArticulatoryWeights(vocalTractMesh, currentWeights, morphIndexMap);
  }
  
  renderer.render(scene, camera);
});
```

The key perceptual point: a minimal **2-contact trill** (the shortest natural `[r]`) spans only ~74ms at 27 Hz. A long emphatic trill can have 6–8 contacts (~220–300ms). Your UI should accept a `contactCount` parameter and compute `trillDuration = contactCount / TRILL_FREQUENCY_HZ` accordingly.

## Output 3 — Waveguide Area Functions

### Theoretical Basis

The vocal tract is modeled as a **1D acoustic tube** of approximately 17.5 cm (male) discretized into **44 cylindrical sections** each ~4 mm long (following the Pink Trombone / Maeda 1979 / Story 1995 parameterization). Each element stores a **diameter in cm**. The cross-sectional area `A = π(d/2)²`. Acoustic resonances (formants F1, F2, F3) emerge from the interference pattern along this tube — a narrow section near a pressure antinode raises a formant frequency; a narrow section near a velocity antinode lowers it.

Index `0` = **glottis** (posterior), index `43` = **lips** (anterior).

### Animating the Trill Area Function

```typescript
// Interpolate between contact and open phases using the trill oscillator output
function trillAreaFunction(timeSeconds: number, trillAge: number): Float32Array {
  const t = trillTTE(timeSeconds, trillAge); // reuse from Output 2
  // t near 1.0 = contact phase; t near 0.5 = open phase
  const blend = (t - 0.50) / (1.0 - 0.50); // map [0.5, 1.0] → [0, 1]
  const clampedBlend = Math.max(0, Math.min(1, blend));
  
  const contact = AREA_FUNCTIONS["spanish_r_contact"];
  const open    = AREA_FUNCTIONS["spanish_r_open"];
  const result  = new Float32Array(44);
  for (let i = 0; i < 44; i++) {
    result[i] = open[i] + (contact[i] - open[i]) * clampedBlend;
  }
  return result;
}
```

### Rendering the Area Function as a Beautiful UI Curve

The mapping from the 44-diameter array to a visual curve works as follows. The tract runs left-to-right (posterior→anterior), and the diameter at each index is plotted symmetrically above and below a centerline to show the tube cross-section. In SVG/Canvas on a `width=600, height=200` canvas:

```typescript
function areaFunctionToSVGPath(
  diameters: Float32Array,
  canvasW = 600,
  canvasH = 200,
  maxDiam = 5.0   // cm — normalize against this
): { upper: string; lower: string } {
  const N    = diameters.length; // 44
  const midY = canvasH / 2;
  const xStep = canvasW / (N - 1);
  
  const upperPoints: string[] = [];
  const lowerPoints: string[] = [];
  
  for (let i = 0; i < N; i++) {
    const x = i * xStep;
    const halfH = (diameters[i] / maxDiam) * (canvasH * 0.45);
    upperPoints.push(`${x.toFixed(1)},${(midY - halfH).toFixed(1)}`);
    lowerPoints.push(`${x.toFixed(1)},${(midY + halfH).toFixed(1)}`);
  }
  
  // Convert to smooth SVG path using cardinal spline approximation
  // (catmull-rom through each point gives organic, non-angular contour)
  return {
    upper: pointsToSmoothPath(upperPoints),
    lower: pointsToSmoothPath(lowerPoints),
  };
}

// Catmull-Rom → cubic Bézier conversion for smooth SVG paths
function pointsToSmoothPath(pts: string[]): string {
  const p = pts.map(s => s.split(',').map(Number) as [number, number]);
  let d = `M ${p[0][0]} ${p[0][1]}`;
  const alpha = 0.5; // tension
  for (let i = 0; i < p.length - 1; i++) {
    const p0 = p[Math.max(0, i - 1)];
    const p1 = p[i];
    const p2 = p[i + 1];
    const p3 = p[Math.min(p.length - 1, i + 2)];
    // Catmull-Rom control points
    const cp1x = p1[0] + (p2[0] - p0[0]) * alpha / 3;
    const cp1y = p1[1] + (p2[1] - p0[1]) * alpha / 3;
    const cp2x = p2[0] - (p3[0] - p1[0]) * alpha / 3;
    const cp2y = p2[1] - (p3[1] - p1[1]) * alpha / 3;
    d += ` C ${cp1x.toFixed(2)} ${cp1y.toFixed(2)}, ${cp2x.toFixed(2)} ${cp2y.toFixed(2)}, ${p2[0]} ${p2[1]}`;
  }
  return d;
}
```

```typescript
function velocityToColor(
  diameter: number,
  maxDiam: number = 5.0,
  minDiam: number = 0.1
): string {
  // Normalized constriction: 0 = wide, 1 = maximally narrow
  const constriction = 1 - Math.max(0, (diameter - minDiam) / (maxDiam - minDiam));
  
  // Map to amber-coral-red ramp for turbulence hotspot visualization

  const b = Math.round(40  - constriction * 40);
  return `rgba(${r},${g},${b},${0.15 + constriction * 0.30})`;
}
```

### Key Integration Notes for Your TypeScript App

**Coordinate system consistency:** All three outputs share the same posterior→anterior orientation. Your Three.js scene Z-axis should map to the SVG X-axis (negative Z = posterior, positive Z = anterior). The area function index `i` maps to `z = -8.75 + i * 0.4` cm in scene space.

**Transition timing:** Human articulatory transitions between sustained phonemes average 80–120ms for tongue body movements and 50–80ms for tongue tip gestures. The velum is slower: 100–200ms for opening (oral → nasal), 150–250ms for closing (nasal → oral). Use these as your lerp durations in the Three.js animation loop for perceptual realism.

**Formant verification:** You can validate your 3D mesh deformations by running the area function arrays through a simple Kelly-Lochbaum tube junction model — formant peaks from the transfer function should land within ±50 Hz of the tabulated values shown in the widget. This gives you a physics-based sanity check on whether your vertex deformations are anatomically plausible.
